   ```typescript
   useEffect(() => {
     if (!open) return
     function handleOutsideClick(event: MouseEvent) {
       if (containerRef.current && !containerRef.current.contains(event.target as Node)) {
         setOpen(false)
       }
     }
     document.addEventListener('mousedown', handleOutsideClick)
     return () => {
       document.removeEventListener('mousedown', handleOutsideClick)
     }
   }, [open])
   ```

Replace the `style` prop value from a fixed `height` to a `maxHeight`:

Before:
```typescript
style={{ height: `calc(100vh - 220px)` }}
```

After:
```typescript
style={{ maxHeight: 'calc(100dvh - 200px)' }}
```

Explanation of the value:
- `100dvh` — uses the dynamic viewport height unit (correct for mobile browsers where the address bar shrinks/grows); consistent with `h-dvh` already used on the root `<div>` in App.tsx.
- `200px` — conservative bottom offset that accounts for: top safe-area + filter row height (~100px total from top) plus the stats bar height (~100px). The panel opens *below* the trigger button, so the 200px offset ensures it does not reach the stats bar on the smallest common viewport (375px height). On larger screens the panel simply won't fill to max-height.
- Switching from `height` to `maxHeight` means the panel shrinks naturally when content is short, rather than always occupying the full calculated space.

Also verify that the outer container div already has `overflow-hidden` (it does — `className="absolute flex flex-col z-20 mt-2 right-0 w-50 rounded-md border border-gray-400 bg-white shadow-map overflow-hidden"`).
