## Interaction Contract

This section is specific to Phase 2 (animation and state transitions are the primary deliverable).

### Drawer States

| State | Drawer Transform | Backdrop Opacity | Backdrop Pointer Events |
|-------|-----------------|------------------|------------------------|
| Closed | `-translate-x-full` (off-screen left) | `opacity-0` | `pointer-events-none` |
| Open | `translate-x-0` (fully visible) | `opacity-100` | `pointer-events-auto` |

### Transition Spec

| Property | Value | Applies To |
|----------|-------|-----------|
| Duration | 300ms (`duration-300`) | Both drawer and backdrop |
| Easing | ease-in-out (`ease-in-out`) | Both drawer and backdrop |
| Animated property — drawer | transform (`transition-transform`) | Drawer panel only |
| Animated property — backdrop | opacity (`transition-opacity`) | Backdrop only |

Rationale for 300ms: Matches typical mobile drawer conventions (200–350ms range). Faster than 300ms can feel jarring on slide-in; slower feels sluggish. Assumption A2 from RESEARCH.md — tunable in one place.

### Drawer Geometry

| Property | Value | Rationale |
|----------|-------|-----------|
| Width | `w-72` (288px) | Natural drawer width on desktop |
| Max width | `max-w-[80vw]` | Prevents overflow on phones narrower than 360px |
| Height | `h-full` | Full viewport height |
| Position | `fixed top-0 left-0` | Slides in from the left edge |
| Z-index — panel | `z-[3000]` | Above backdrop, filter controls, and map |
| Z-index — backdrop | `z-[2900]` | Above map (z-[1001]), below drawer panel |
| Overflow | `overflow-y-auto` | Safety measure for future content growth |

Source: RESEARCH.md Architecture Patterns section — z-index values verified against existing codebase usage.

### Close Triggers

| Trigger | Behavior |
|---------|----------|
| Tap close button (X) | Calls `onClose()` → `setShowMenu(false)` in App.tsx |
| Tap backdrop | Backdrop `onClick` calls `onClose()` |
| Tap menu item | Each item calls `onClose()` then its modal handler |

### Mount Strategy

The drawer is always mounted in the DOM (never conditionally rendered). The `open` prop drives visible state via CSS classes. This is required for the exit slide animation (MENU-02) — unmounting removes the element before the transition plays.

App.tsx change: `{showMenu && <HamburgerMenu ... />}` → `<HamburgerMenu open={showMenu} ... />`.

Source: RESEARCH.md Pattern 1 and Pitfall 1.
