# Phase 02.1 Plan 01: Fix Modal Transitions for PrivacyNotice and TrademarkNotice Summary

**One-liner:** Always-mounted bottom-sheet panels with `transition-transform` + `translate-y-0/translate-y-full` slide, matching the HamburgerMenu pattern exactly.

## What Was Built

Both `PrivacyNotice` and `TrademarkNotice` were broken in two ways:
1. They were conditionally mounted (`{showX && <Component />}`), which destroys the component before any exit animation can play.
2. Their animation used `transition-[inset]` + `inset-y-100`/`inset-y-full`, which is not a reliable cross-browser animation approach.

This plan fixes both issues by applying the proven pattern from `HamburgerMenu.tsx`:

- Added `open: boolean` prop to both components
- Added a `transition-opacity` backdrop (same z-[2900] / pointer-events pattern as HamburgerMenu)
- Replaced broken `transition-[inset]` with `transition-transform duration-300 ease-in-out`
- Open state: `translate-y-0` | Closed state: `translate-y-full`
- Added `useRef` / `useEffect` focus management (close button receives focus on open)
- Added `role="dialog" aria-modal="true" aria-label="..."` to both panels
- Removed `delay-150` from TrademarkNotice (was causing panel/backdrop offset)
- Removed `{showPrivacy && ...}` and `{showTrademark && ...}` guards in App.tsx — both panels are always in the DOM

**2. [Rule 2 - Missing Critical Functionality] Added focus management to both panels**
- **Found during:** Task 1 and Task 2
- **Issue:** HamburgerMenu focuses the close button when opened. Without this, keyboard navigation breaks — opening a modal without moving focus is an accessibility gap.
- **Fix:** Added `useRef<HTMLButtonElement>` + `useEffect(() => { if (open) closeButtonRef.current?.focus() }, [open])` to both panels. The `ref` is attached to the Fermer button.
- **Files modified:** src/components/PrivacyNotice.tsx, src/components/TrademarkNotice.tsx
- **Commit:** 24187bf
