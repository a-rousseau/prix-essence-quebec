# Phase 02.1: Fix Modal Transitions for PrivacyNotice and TrademarkNotice - Context

**Gathered:** 2026-04-10
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix the broken slide-in/slide-out transitions on the PrivacyNotice and TrademarkNotice bottom-sheet panels. Both panels currently use a broken animation approach (`transition-[inset]` + `inset-y-100`/`inset-y-full`) and are conditionally unmounted on close, which kills the exit animation entirely.

Scope: Animation fix only. No new features, no copy changes, no layout changes.

</domain>

<decisions>
## Implementation Decisions

### Mount strategy
- **D-01:** Both PrivacyNotice and TrademarkNotice must be **always-mounted** (like HamburgerMenu) — remove the `{showPrivacy && ...}` and `{showTrademark && ...}` guards in App.tsx. Components stay in the DOM at all times; the `open` prop controls position.
- **D-02:** Keep the existing `open: boolean` prop interface — no signature changes.

### Animation approach
- **D-03:** Replace `transition-[inset]` + `inset-y-100`/`inset-y-full` with `transition-transform` + `translate-y-0`/`translate-y-full` — the same pattern HamburgerMenu uses for its horizontal slide (`translate-x-0`/`-translate-x-full`).
- **D-04:** Use the same duration/easing as HamburgerMenu: `duration-300 ease-in-out`.
- **D-05:** Backdrop behavior stays the same (`transition-opacity`, `opacity-100`/`opacity-0`, `pointer-events-auto`/`pointer-events-none`) — this already works correctly.

### Consistency with HamburgerMenu
- **D-06:** The two panels should follow the HamburgerMenu implementation pattern as closely as possible — same backdrop class structure, same transition utilities, same duration.
- **D-07:** Remove the `delay-150` on TrademarkNotice (leftover — panel and backdrop should animate together, not offset).

### Claude's Discretion
- Exact Tailwind utility ordering within class strings
- Whether to extract a shared bottom-sheet wrapper component (acceptable but not required)

</decisions>

<specifics>
## Specific Ideas

- HamburgerMenu (`src/components/HamburgerMenu.tsx`) is the authoritative reference for this pattern. The fix should make PrivacyNotice and TrademarkNotice look structurally identical (but vertically instead of horizontally).
