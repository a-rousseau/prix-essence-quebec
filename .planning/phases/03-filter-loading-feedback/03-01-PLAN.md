Create `src/components/FilterOverlay.tsx` as a new file with the following exact implementation.

The component uses the always-mounted pattern (element stays in DOM; opacity toggled via CSS) so the exit fade-out animation works. Conditional rendering (`{visible && <FilterOverlay />}`) would kill the exit animation by unmounting instantly.

```tsx
interface FilterOverlayProps {
  visible: boolean
}

export function FilterOverlay({ visible }: FilterOverlayProps) {
  return (
    <div
      className={`absolute inset-0 z-[2000] flex items-center justify-center bg-black/40 transition-opacity duration-150 ease-in-out ${visible ? 'opacity-100 pointer-events-auto' : 'opacity-0 pointer-events-none'}`}
      aria-hidden="true"
    >
      <div className="w-10 h-10 border-4 border-white/30 border-t-white rounded-full animate-spin" />
    </div>
  )
}
```

Key decisions implemented:
- D-01: `bg-black/40` (dark, translucent — map remains faintly visible beneath). NOT `bg-white/90` which is used by `LoadingSpinner`.
- D-02: Spinner only. No text label beneath the spinner. No `<p>` element.
- `aria-hidden="true"` — overlay is decorative, not interactive.
- Overlay root classes: `absolute inset-0 z-[2000] flex items-center justify-center bg-black/40 transition-opacity duration-150 ease-in-out` (same z-index as LoadingSpinner; they are mutually exclusive in normal flow).
- Spinner classes: `w-10 h-10 border-4 border-white/30 border-t-white rounded-full animate-spin` — 40px (smaller than LoadingSpinner's 48px `w-12 h-12` to reinforce distinction; white ring against dark background).
- No `backdrop-blur` — dark translucent overlay provides sufficient separation; blur on dark adds noise.
- `pointer-events-none` when hidden prevents phantom click blocking.
- Named export (not default export) per project convention.
- No new npm dependencies.

**Step 4 — Replace the existing filter useEffect:**

```tsx

  setFilteredStations(filterStations(stations, filterState))
```

Replace it entirely with:
```tsx
useEffect(() => {
  // Guard: don't show filter overlay before stations data arrives.
  // LoadingSpinner handles the initial load state.
  if (stations.length === 0) {
    setFilteredStations([])
    return
  }

  // Show overlay and record start time (D-03, D-04)
  setFilterPending(true)
  overlayStartRef.current = Date.now()

  // Cancel any pending hide timer to handle rapid filter changes
  if (overlayTimerRef.current !== null) {
    clearTimeout(overlayTimerRef.current)
    overlayTimerRef.current = null
  }

  // Apply filter synchronously
  const result = filterStations(stations, filterState)
  setFilteredStations(result)

  // Enforce 150ms minimum before hiding overlay (D-03)
  const elapsed = Date.now() - overlayStartRef.current
  const remaining = Math.max(0, 150 - elapsed)
  overlayTimerRef.current = setTimeout(() => {
    setFilterPending(false)
    overlayTimerRef.current = null
  }, remaining)

  // Cleanup: cancel pending timer on unmount or next effect run
  return () => {
    if (overlayTimerRef.current !== null) {
      clearTimeout(overlayTimerRef.current)
    }
  }
}, [stations, filterState])
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filterState → filterStations | User-supplied filter values flow into a pure JS function operating on in-memory data; no network, no DOM injection |
| filterPending state → JSX | Boolean state drives CSS class toggling; no user data is interpolated into class strings |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Denial of Service | App.tsx rapid filter toggling | mitigate | `clearTimeout` before each new timer prevents stacked `setTimeout` callbacks from accumulating; the overlay lifecycle always resolves within one 150ms window after the last change |
| T-03-02 | Tampering | FilterOverlay.tsx — CSS class interpolation | accept | `visible` prop is a `boolean` (TypeScript-enforced); no user string is interpolated into the className. XSS via class injection is not possible. |
| T-03-03 | Information Disclosure | filterPending state | accept | `filterPending` is a local boolean with no PII; toggling it exposes no sensitive information |
| T-03-04 | Elevation of Privilege | overlay pointer-events | mitigate | `pointer-events-none` applied when overlay is hidden (`opacity-0`) prevents phantom click blocking on underlying controls; `aria-hidden="true"` removes overlay from accessibility tree when visible |
</threat_model>
