### Integration Defects

#### DEFECT-01 — SUMMARY/code mismatch on overlay duration (info)
**Affected REQ:** FILTER-01, FILTER-02
Phase 03 SUMMARY describes `Math.max(0, 150 - elapsed)` logic measuring elapsed time. Live code uses `FILTER_OVERLAY_MIN_MS = 500` with a flat `setTimeout` — no elapsed measurement. The VERIFICATION.md captures the user-approved 150ms→500ms change but the SUMMARY was written before the adjustment. **No functional break.**

#### DEFECT-02 — Stale `inset-100` class (tech_debt)
**Affected REQ:** None directly
`PrivacyNotice.tsx:29` and `TrademarkNotice.tsx:28` contain `open ? 'translate-y-0 inset-100' : 'translate-y-full'`. The `inset-100` class was supposed to be removed as part of Phase 02.1 per the VERIFICATION.md gaps. User accepted the deviation after confirming animation worked visually. Dead code; `fixed bottom-0 left-0 right-0` positioning overrides any inset. **No visual break.**

#### DEFECT-03 — Shared z-[2000] with DOM-order stacking (tech_debt)
**Affected REQ:** FILTER-01, FILTER-02
FilterOverlay, LoadingSpinner, and error banner all use `z-[2000]`. Correct stacking (error banner visible above overlay) works because error banner appears later in the DOM (line 114 vs 111). Any DOM reorder would break this. **No current break; fragile pattern.**
